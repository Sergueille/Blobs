How to make levels?

This page will show you how to create your own levels for the game. I'm too lazy to make a level editor, so you will have to use text files...

Creating the level file

In the game, click on the my levels button. The directory where the level files are located is shown on the top of the screen.
Create a text file in this folder, with extension .txt

File header

At the top of the file, you must include this header:

<Collection name>
<Description>
-
  • Collection name is the name of the level collection, it can be different than the name of the file
  • Description is a short description of the collection (only one line)
Don't forget the -! This character is the separator.

Adding a level

After the header, you can add as many levels as you want. Here is a level template:

<Level name>
###b
#.!.
a..#
- <Move count>
a blob red 1
b end red 1
-
  • Level name is the name of the level that will be shown ingame. It can contains special characters, but not for the first letter.
  • Move count is the minimum number of moves required to complete the level (displayed on top of the screen)
  • On the next lines is the level map. Each character represents a tile. The map must be rectangular.
    • . is a white tile, where objects can move
    • # is outside the level, where objects cannot move
    • ! is a sticky diamond
    • ? is a color inverter
    • Characters form a to z are other objects, that should be described under the map
  • After the separator are the object descriptions. Each line represents one object:
    <Character of the object> <Type> <Color> <Number of eyes>
    • Character of the object is the character used on the map for this object
    • Type is blob or end
    • Color is red, blue, green, cyan, magenta, yellow or brown (or none for eyes without blob)
    • Number of eyes is an integer
Any additional whitespace or blank lines will be ignored (hopefully, there might be some bugs related to that)
Again, don't forget the -, even if it's the last level!
On the last level, you should also add at least one blank line after the last separator

We're finished!

You can now open the level file in the game! (You may have to close and open the game to refresh the level list).

Common errors

In level map Title at position (x, y): expected a map character, got something.

You may have forgot the - at the end of the map (shame on you), and the game thinks the object descriptions are inside the map, and is very confused.
You may also have included a bad character in the map. You can only use ., #, !, ?, or characters between a to z.

In level objects Title: the object something is not present in the map or have been described twice
In level objects Title: there are x objects on map, bun only y objects described

An object is described but not in the map, or on the map but not described, or is described twice, or maybe present twice in the map.

Expected a character, but got to end of file

There is a high probability that you forgot the - at the end, you fool!
Maybe you also forgot the blank line at the very end of the file.
